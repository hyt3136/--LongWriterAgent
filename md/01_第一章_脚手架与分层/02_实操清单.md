# 第1章实操清单

## A. 运行与观察

1. 进入 code 目录。
2. 设置 PYTHONPATH=src。
3. 运行健康检查命令并观察 JSON 输出字段。
4. 运行 unittest，确认基础测试通过。

## B. 练习任务（你来做）

1. 在配置中新增 REQUEST_TIMEOUT_MS。
2. 约束范围：100 到 120000。
3. 健康检查输出新增 request_timeout_ms 字段。
4. 新增 2 条测试：
   - 默认值测试
   - 非法值抛错测试

## C. 评审标准（我会按这个给你打分）

1. 是否遵守分层边界。
2. 是否在 config 层完成校验，而不是在业务层补丁式判断。
3. 是否补齐测试而不是仅修改运行逻辑。
4. 错误信息是否可读。
